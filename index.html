1. Introducción

En este laboratorio se ha implementado una base de datos relacional para una clínica, aplicando el concepto de Vistas (Views) en Oracle SQL para simplificar consultas y mejorar la seguridad de los datos.

2. Scripts de Creación y Vistas

A continuación se presenta el código principal utilizado para resolver los ejercicios propuestos en la guía:

-- Ejercicio 5.1: Creación de Vista Médicos y Especialidades
CREATE OR REPLACE VIEW v_Medicos_Especialidades AS
SELECT M.nombresM, M.paternoM, E.nombreEsp
FROM tMedico M
JOIN tEspecialidadDelMedico EM ON M.codM = EM.codM
JOIN tEspecialidad E ON EM.codEsp = E.codEsp;

-- Ejercicio 5.4: Vista de Enfermedades No Detectadas
CREATE OR REPLACE VIEW v_Enfermedades_No_Detectadas AS
SELECT nombreE FROM tEnfermedad
WHERE codE NOT IN (SELECT codE FROM tEnfermedadEnElDiagnostico);

4. Conclusiones

Impacto de las Vistas: Las vistas facilitan el acceso a la información al ocultar la complejidad de los JOINS. Permiten que los usuarios finales consulten datos consolidados sin necesidad de conocer la estructura interna de las tablas, optimizando el tiempo de respuesta y garantizando que la información no se repita innecesariamente.
